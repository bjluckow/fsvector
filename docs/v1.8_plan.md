# fsvector v1.8 — Polishing & Optimization

## Overview

v1.8 focuses on search quality, indexing performance, and operational
polish. No new modalities or services. The goal is to make fsvector
production-ready for the S3 indexing run and fssorter.

## M1.8.1 — Search Modes & Tuning

### Search modes

`search.SearchQuery` gains a `Mode` field:
```go
type SearchMode string

const (
    SearchModeHybrid   SearchMode = "hybrid"    // default: semantic + FTS
    SearchModeVector   SearchMode = "vector"     // pure cosine similarity
    SearchModeFullText SearchMode = "fulltext"   // FTS only, no vector
)

type SearchQuery struct {
    Query    string
    Vector   []float32
    Mode     SearchMode  // default: hybrid
    Limit    int
    Offset   int
    // filters unchanged
    Modality string
    Ext      string
    Source   string
    Since    *time.Time
    Before   *time.Time
    MinSize  *int64
    MaxSize  *int64
    MinScore *float64
}
```

### Score expression per mode
```go
// vector mode — pure cosine similarity
scoreExpr = `1 - (embedding <=> $1)`

// hybrid mode — semantic + FTS, capped at 1.0
scoreExpr = `
    LEAST(1.0,
        $semantic_weight * (1 - (embedding <=> $1)) +
        $fts_weight * CASE
            WHEN <fts_subquery> = 0 THEN 0.0
            ELSE GREATEST($fts_min_boost, LEAST(1.0, <fts_subquery> * $fts_scale))
        END
    )`

// fulltext mode — FTS only, no vector required
scoreExpr = `
    CASE
        WHEN <fts_subquery> = 0 THEN 0.0
        ELSE LEAST(1.0, <fts_subquery> * $fts_scale)
    END`
```

Full-text mode doesn't require a query vector — useful for exact
keyword searches where semantic similarity is irrelevant.

### Tuning parameters

Configurable via `fsvector.yaml` and overridable per-request:
```yaml
search:
  semantic_weight: 0.5      # weight for vector similarity component
  fts_weight: 0.5           # weight for FTS component
  fts_scale: 10.0           # multiplier to bring ts_rank into [0,1] range
  fts_min_boost: 0.3        # minimum FTS bonus for any keyword match
  default_mode: hybrid      # hybrid | vector | fulltext
```

These flow into `search.SearchConfig`:
```go
type SearchConfig struct {
    SemanticWeight float64
    FTSWeight      float64
    FTSScale       float64
    FTSMinBoost    float64
    DefaultMode    SearchMode
}
```

`SearchConfig` is loaded once in `daemon/server.go` from config and
passed to `search.Search`. Per-request overrides can be added to
`api.SearchRequest`:
```go
type SearchRequest struct {
    // existing fields...
    Mode           string  `json:"mode"`            // "hybrid" | "vector" | "fulltext"
    SemanticWeight float64 `json:"semantic_weight"` // 0 = use config default
    FTSWeight      float64 `json:"fts_weight"`
}
```

### Score capping

Wrap the entire hybrid score expression in `LEAST(1.0, ...)` to
prevent scores above 1.0. This was deferred from v1.7.

### SearchQuery.Mode in fsvector CLI
```bash
fsvector search "roof damage" --mode vector      # pure semantic
fsvector search "roof damage" --mode hybrid      # default
fsvector search "roof damage" --mode fulltext    # keyword only
```

**Verify:**
```bash
# hybrid (default)
./bin/fsvector search "dog in a field" --modality image
# → dog.webp ranks high, scores in [0,1]

# vector only
./bin/fsvector search "dog in a field" --modality image --mode vector
# → same ranking but no FTS boost

# fulltext only
./bin/fsvector search "best of times" --modality image --mode fulltext
# → bestoftimes.png ranks high via OCR match
# → files with no text_content score 0

# score always in [0,1]
./bin/fsvector search "best of times"
# → no score above 1.0
```

## M1.8.2 — Image Similarity Search

### `fsvector search --image`
```bash
fsvector search --image /path/to/query.jpg
fsvector search --image /path/to/query.jpg --modality image
fsvector search --image s3://bucket/key.jpg  # future
```

When `--image` is provided instead of a text query:
1. Read image bytes from local path
2. POST to `/embed/image` → get embedding vector
3. Search with that vector, `Mode=vector` (no FTS — no text query)
4. Return visually similar files

### CLI changes
```go
var searchImagePath string

searchCmd.Flags().StringVar(&searchImagePath, "image", "", 
    "path to image file for visual similarity search")
```

In `searchCmd.RunE`:
```go
var req api.SearchRequest

if searchImagePath != "" {
    // image search
    data, err := os.ReadFile(searchImagePath)
    if err != nil {
        return fmt.Errorf("read image: %w", err)
    }
    filename := filepath.Base(searchImagePath)
    embResp, err := client().EmbedImage(ctx, filename, data)
    if err != nil {
        return fmt.Errorf("embed image: %w", err)
    }
    req = api.SearchRequest{
        // no Query — image search uses vector directly
        Mode:     "vector",
        Limit:    searchLimit,
        Page:     searchPage,
        Modality: searchModality,
        Ext:      searchExt,
    }
    // pass pre-computed vector to search
    // requires new api.SearchRequest field or separate endpoint
} else {
    req = api.SearchRequest{
        Query: args[0],
        // ...
    }
}
```

}
