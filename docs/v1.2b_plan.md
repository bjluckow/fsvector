# fsvector v1.2b Plan — Multimodal Embeddings (Clean)

## Context

v1.2 mixed two concerns: multimodal embedding support and hot-swapping
models. Hot-swapping adds significant infrastructure complexity that is
not needed to achieve the primary goal of indexing and classifying a
large multimodal file collection. v1.2b preserves everything useful from
v1.2 and removes all hot-swapping infrastructure.

Hot-swapping is deferred entirely. When it returns it will be a
dedicated version with a focused plan.

---

## What v1.2 Built (Keeping)

- Two embedding services: `embedsvc-text` and `embedsvc-image`
- Typed Go clients: `embed.TextClient` and `embed.ImageClient`
- Per-modality embedding columns: `text_embedding` and `image_embedding`
- Separate HNSW indexes per modality
- `pipeline/mappings.go` — single `fileTypes` map owning all routing
- `embed/common.go` — shared `HealthResponse`, `HotswapTimeout`
- Two-dim schema migration with `%%TEXT_DIM%%` and `%%IMAGE_DIM%%`
- Config split: `TEXT_EMBEDSVC_URL`, `IMAGE_EMBEDSVC_URL`,
  `TEXT_EMBED_MODEL`, `IMAGE_EMBED_MODEL`

---

## What v1.2b Removes

### Schema
- `retired_at` column on `files`
- All indexes filtering on `retired_at IS NULL`
- `models` table
- `files_embed_model_fk` foreign key constraint
- `schema_version` table (reverted to simple idempotent migration)

### Store
- `store/registry.go` — entire file removed
- `store/retire.go` — entire file removed
- `store/migrate.go` — revert to single idempotent `Migrate(ctx, conn,
  textDim, imageDim)` with no version tracking

### Search
- Remove `retired_at IS NULL` from all WHERE clauses in `search/`

### Embed
- `embed/common.go` — remove `HotswapTimeout` (no swap endpoint needed)
- `embed/text.go` — remove `SwapModel()`
- `embed/image.go` — remove `SwapModel()`

### embedsvc
- `services/embedsvc/main.py` — remove `POST /model` endpoint,
  `_loading` flag, `_lock`, `last_error` — revert to simple
  single-model service that loads on startup and serves forever

### fsvectord
- Remove `RegisterModel` calls from startup
- Remove all hot-swap and retire logic

### docs
- `v1_2_plan.md` — kept for reference but superseded by this doc

---

## Simplified embedsvc

Single model, loads on startup, serves forever. No swap endpoint,
no loading state, no lock.

---

## Simplified fsvectord Startup
```
1. Load config
2. Connect to embedsvc-text → get model name + dim
3. Connect to embedsvc-image → get model name + dim
4. Connect to postgres
5. Migrate schema (idempotent, no versioning needed)
6. Reconcile filesystem
7. Start fsnotify watcher
```

No model registration, no registry checks, no retire logic.

## Milestones

### M1.2b.1 — Schema cleanup
**Goal:** Remove `retired_at`, `models` table, FK constraint,
`schema_version` table from schema. Revert to simple idempotent
migration. Wipe and rebuild DB.

**Verify:**
```bash
docker compose down -v && docker compose up -d postgres
# fsvectord applies clean schema, no version table, no models table
docker compose exec postgres psql -U fsvector -c "\d files"
# → no retired_at column
# → text_embedding and image_embedding present
```

---

### M1.2b.2 — Code cleanup
**Goal:** Remove all hot-swap code from Go and Python.

Files changed:
- `store/registry.go` → deleted
- `store/delete.go` → remove RetireByModality, UnRetire
- `store/migrate.go` → revert to simple single function
- `embed/common.go` → remove HotswapTimeout
- `embed/text.go` → remove SwapModel
- `embed/image.go` → remove SwapModel
- `search/*.go` → remove retired_at IS NULL filters
- `services/embedsvc/main.py` → revert to simple version
- `cmd/fsvectord/main.go` → remove RegisterModel calls

**Verify:**
```bash
make build
go test ./...
DATABASE_URL=... ./bin/fsvectord
# → clean startup, reconcile runs, files indexed
fsvector search "resume"
fsvector search "dog" --modality image
fsvector stats
```

---

### M1.2b.3 — End-to-end verification
**Goal:** Full multimodal indexing working cleanly with both services.
Text files go to `embedsvc-text`, images go to `embedsvc-image`.
Search returns results from the correct embedding column.

**Verify:**
```bash
fsvector search "quarterly report" --modality text
# → text results only, scores from text_embedding column

fsvector search "a dog running outside" --modality image
# → image results only, scores from image_embedding column

fsvector stats
# → correct text/image file counts
```
