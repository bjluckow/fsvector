# fsvector v1.2 Plan — Multi-Model & Hot-Swapping

## Goals

Introduce support for multiple embedding models, one per modality, with
first-class hot-swapping via a CLI command. Separate embedding services
per modality in docker-compose. Add a model registry to track all models
that have ever been used, enabling future comparison and a model manager
UI. Extend the schema to cleanly distinguish filesystem-driven soft-deletes
from model-swap retirements. Move all file type routing and conversion
decisions into `pipeline/`, with `embed/` providing typed clients per
modality.

---

## Design Decisions

### One model per modality, one service per modality

Each modality (`text`, `image`) gets its own dedicated embedding service
in docker-compose. This means:

- Text and image models are independently configurable and restartable
- Swapping the image model does not affect text indexing
- Scales naturally to `embedsvc-audio` and `embedsvc-video` in v1.5

Services:
```
embedsvc-text    loads TEXT_EMBED_MODEL on startup
embedsvc-image   loads IMAGE_EMBED_MODEL on startup
```

---

### Typed embed clients per modality

`embed/` provides one typed client per modality, each shaped around
what that modality actually returns. There is no generic `embed.Client`
or router — each client is purpose-built:

```
internal/embed/
├── text.go     # TextClient  — EmbedTexts() → [][]float32
└── image.go    # ImageClient — EmbedImage() → []float32
```

Video and audio will add their own typed clients in v1.5:
```
├── video.go    # VideoClient — EmbedFrames() → []FrameEmbedding
└── audio.go    # AudioClient — EmbedTranscript() → [][]float32
```

Each client is constructed with a base URL pointing at its respective
service:

```go
textClient  := embed.NewTextClient("http://embedsvc-text:8000")
imageClient := embed.NewImageClient("http://embedsvc-image:8000")
```

The `/model` swap endpoint is implemented per client:

```go
func (c *TextClient)  SwapModel(ctx context.Context, model string) (*HealthResponse, error)
func (c *ImageClient) SwapModel(ctx context.Context, model string) (*HealthResponse, error)
```

---

### pipeline/ owns all routing and conversion decisions

`pipeline/` is the single place that decides:

1. Which modality a file belongs to (ext → modality mapping)
2. Whether conversion is needed before embedding (ext → conversion mapping)
3. Which typed embed client to call

These are two distinct mappings that are related but separate:

```
ext → modality    (.pdf → text, .heic → image, .jpg → image)
ext → conversion  (.pdf → pdftotext, .heic → imagemagick, .txt → none)
```

A file can require conversion AND belong to a modality independently:
- `.pdf`  → modality=text,  needs conversion (pdftotext)
- `.heic` → modality=image, needs conversion (imagemagick)
- `.txt`  → modality=text,  no conversion
- `.jpg`  → modality=image, no conversion

Both maps live in `internal/pipeline/mappings.go` as package-level vars.
`embed/` has no knowledge of file types or routing — it only handles
HTTP calls to the embedding services.

`pipeline.Config` holds typed clients directly:

```go
type Config struct {
    TextEmbed     *embed.TextClient
    ImageEmbed    *embed.ImageClient
    ConvertClient *convert.Client
    Source        string
    MinEmbedSize  int64
}
```

### Model registry (store/registry.go)

A `models` table tracks every model that has ever been used per modality.
This is a registry, not just an operational lock — it persists historical
model usage and enables future comparison queries and a model manager UI.

The registry lives in `store/registry.go` since it is a database write
concern, keeping `embed/` free of any DB dependency.

```sql
CREATE TABLE models (
    modality    TEXT NOT NULL,
    model_name  TEXT NOT NULL,
    dim         INT NOT NULL,
    status      TEXT NOT NULL DEFAULT 'ready',
    -- 'ready'      : active and accepting queries
    -- 'reindexing' : hot-swap in progress, modality is locked
    -- 'retired'    : superseded by a newer model for this modality
    is_active   BOOLEAN NOT NULL DEFAULT false,
    indexed_at  TIMESTAMPTZ,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    PRIMARY KEY (modality, model_name)
);

-- enforce only one active model per modality at a time
CREATE UNIQUE INDEX models_active_modality_idx
    ON models (modality)
    WHERE is_active = true;
```

On `fsvectord` startup, the active model for each modality is registered
if not already present. If the configured model differs from the active
one in the registry, `fsvectord` logs a warning and suggests running
`fsvector model swap`.

### Schema changes to files table

#### New unique key

The unique index on `files` changes from `(path, chunk_index)` to
`(path, chunk_index, embed_model)`. This allows multiple embeddings of
the same file to coexist under different models, enabling caching of
old embeddings for swap-back and historical comparison.

```sql
DROP INDEX IF EXISTS files_path_chunk_idx;

CREATE UNIQUE INDEX files_path_chunk_model_idx
    ON files (path, chunk_index, embed_model);
```

#### New retired_at column

A `retired_at` column distinguishes model-swap retirements from
filesystem soft-deletes. These are fundamentally different events:

```sql
retired_at  TIMESTAMPTZ  -- NULL = active embedding
                         -- non-NULL = superseded by a model swap
```

| Column | Set by | Meaning |
|---|---|---|
| `deleted_at` | fsvectord watcher / reconcile | File no longer exists on the watched filesystem |
| `retired_at` | Hot-swap process | Embedding is stale, superseded by a newer model |

**The filesystem is the source of truth.** A file with `deleted_at` set
must never appear in any query under any circumstances — active search,
historical model comparison, model manager stats, or purge candidates.
`deleted_at` always takes absolute precedence over `retired_at`.

#### Query filter rules

| Query type | SQL filter |
|---|---|
| Active search | `WHERE deleted_at IS NULL AND retired_at IS NULL` |
| Historical comparison | `WHERE deleted_at IS NULL AND retired_at IS NOT NULL AND embed_model = $1` |
| Model manager stats | `WHERE deleted_at IS NULL GROUP BY embed_model, modality` |
| Purge candidates | `WHERE retired_at IS NOT NULL AND embed_model = $1` |

Note: purge operations never touch rows where `deleted_at IS NOT NULL` —
filesystem state is always preserved separately from model state.

---

### Row caching on model swap-back

When swapping back to a previously used model, retired rows for that
model are un-retired rather than re-embedded, provided the file's
`content_hash` has not changed. This makes swap-back fast and free,
and means the index functions as a cache of previous model results.

Swap-back logic per file:
1. Check for existing row with `(path, chunk_index, embed_model)` where
   `retired_at IS NOT NULL` and `deleted_at IS NULL`
2. If found and `content_hash` matches: clear `retired_at`, skip embed
3. If found and `content_hash` differs: re-embed, update row
4. If not found: embed from scratch, insert new row

---

### Hot-swap process

Hot-swap is triggered by `fsvector model swap <modality> <model-name>`.

```
1. Validate: confirm model name is reachable (pre-flight check)

2. LOCK modality in models table (status = 'reindexing')
   → active search continues returning current results until step 6

3. POST /model {"model": "<model-name>"} to the relevant embedsvc
   → service unloads current model, loads new one
   → /embed/* returns 503 while loading
   → /health returns {"status": "loading"} during this window
   → blocks until ready, returns new dim

4. Register new model in models table if not already present

5. Soft-retire all active rows for this modality:
   UPDATE files
   SET retired_at = now()
   WHERE modality = $1
     AND retired_at IS NULL
     AND deleted_at IS NULL

6. UNLOCK: mark old model as retired (is_active=false, status='retired'),
   mark new model as active (is_active=true, status='ready')
   → from this point active search uses the new model

7. Re-embed all files for this modality:
   → for each file: apply swap-back cache logic (see above)
   → upsert new embeddings under new model name

8. Update models table: indexed_at = now()

9. Log: "reindexed 247 image files with <model-name>"
```

**Crash recovery:** If `fsvectord` crashes during step 7, on restart it
detects `status = 'reindexing'` in the models table and resumes. Files
already re-embedded have a live row under the new model; remaining files
still have only retired rows and will be re-embedded.

### embedsvc /model endpoint

Both `embedsvc-text` and `embedsvc-image` gain a `/model` endpoint:

```
POST /model
{"model": "sentence-transformers/all-MiniLM-L6-v2"}

→ 200 OK (after model finishes loading)
{"status": "ok", "model": "...", "dim": 384}

GET /health (during load)
{"status": "loading", "model": "...", "dim": 0}
```

---

### fsvector model subcommands

```
fsvector model list
  → lists all models in the registry with status, dim, file count,
    and indexed_at per modality

fsvector model swap <modality> <model-name>
  → triggers full hot-swap for the given modality

fsvector model status
  → shows current active model per modality
  → shows reindexing progress if a swap is in progress

fsvector model purge <model-name>
  → hard-deletes all retired rows for a given model name
  → only affects rows where retired_at IS NOT NULL
  → never touches rows where deleted_at IS NOT NULL
  → requires confirmation prompt
```

## Schema Migration

v1.2 requires a migration rather than a full schema recreation since
production data exists. A `schema_version` table is introduced:

```sql
CREATE TABLE IF NOT EXISTS schema_version (
    version     INT NOT NULL,
    applied_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

`store/migrate.go` checks the current version and applies pending
migrations in order. The v1.2 migration:

```sql
-- v1.2
ALTER TABLE files ADD COLUMN IF NOT EXISTS retired_at TIMESTAMPTZ;
DROP INDEX IF EXISTS files_path_chunk_idx;
CREATE UNIQUE INDEX IF NOT EXISTS files_path_chunk_model_idx
    ON files (path, chunk_index, embed_model);
CREATE TABLE IF NOT EXISTS models ( ... );
CREATE UNIQUE INDEX IF NOT EXISTS models_active_modality_idx
    ON models (modality) WHERE is_active = true;
INSERT INTO schema_version (version) VALUES (2);
```
