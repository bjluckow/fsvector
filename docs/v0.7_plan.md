# fsvector v0.7 — S3 Support, Source Abstraction & Daemon API

## Overview

v0.7 introduces S3 as a file source alongside the existing local
filesystem, abstracts file reading behind a `FileReader` interface,
makes filesystem watching optional, and exposes a local HTTP API on
`fsvectord` for programmatic control. Email parsing is added as a new
pipeline modality — `.eml` files are just files like any other.

---

## Goals

1. Index files from S3 buckets without requiring local storage
2. Clean `FileReader` abstraction so pipeline is source-agnostic
3. Watching is opt-in — S3 sources reconcile on demand or via polling
4. `fsvectord` exposes a local HTTP API for `fsvector reconcile` and
   future web UI use (now known as 'reindexing')

---

## FileReader Interface

Lives in `internal/source/reader.go`. Used by pipeline functions
instead of `os.ReadFile` directly.
```go
package source

type FileReader interface {
    // Read returns the full contents of a file.
    // For large files, implementations may buffer to a temp file
    // and stream — callers should not assume in-memory storage.
    Read(ctx context.Context, path string) ([]byte, error)

    // Exists returns true if the file exists at path.
    // For local: os.Stat. For S3: HeadObject.
    Exists(ctx context.Context, path string) (bool, error)
}
```

### LocalReader
```go
// source/local_reader.go
type LocalReader struct{}

func (r *LocalReader) Read(ctx context.Context, path string) ([]byte, error) {
    return os.ReadFile(path)
}

func (r *LocalReader) Exists(ctx context.Context, path string) (bool, error) {
    _, err := os.Stat(path)
    if os.IsNotExist(err) {
        return false, nil
    }
    return err == nil, err
}
```

### S3Reader
```go
// source/s3_reader.go
type S3Reader struct {
    Client          *s3.Client
    Bucket          string
    LargeFileBytes  int64  // threshold for temp file vs in-memory
}

func (r *S3Reader) Read(ctx context.Context, path string) ([]byte, error)
func (r *S3Reader) Exists(ctx context.Context, path string) (bool, error)
```

For files over `LargeFileBytes` (default 100MB), `Read` downloads
to a temp file, reads it, and deletes it. Under the threshold,
streams directly to `[]byte`.

## Source Interface

Lives in `internal/source/source.go`.
```go
package source

// FileInfo is source-agnostic file metadata.
// Replaces fsindex.FileInfo as the canonical type passed to pipeline.
type FileInfo struct {
    Path       string
    Name       string
    Ext        string
    Size       int64
    MimeType   string
    Hash       string
    ModifiedAt time.Time
    CreatedAt  time.Time
    SourceURI  string  // "local" or "s3://bucket/key"
}

// Event represents a filesystem or source change event.
type Event struct {
    Kind EventKind
    Path string
}

type EventKind int
const (
    EventCreate EventKind = iota
    EventUpdate
    EventDelete
)

// Watcher is implemented by sources that support live event watching.
// Sources that don't support watching (e.g. S3) do not implement this.
type Watcher interface {
    Watch(ctx context.Context, events chan<- Event) error
}

// Source abstracts a file collection — local filesystem or S3.
type Source interface {
    // Walk returns all files in the source.
    Walk(ctx context.Context) ([]FileInfo, error)

    // Reader returns a FileReader for this source.
    Reader() FileReader

    // URI returns the source identifier stored in the DB.
    // e.g. "local" or "s3://bucket/prefix"
    URI() string
}
```

`LocalSource` implements `Source` + `Watcher`.
`S3Source` implements `Source` only.

`fsvectord` checks for `Watcher` after reconciliation:
```go
if w, ok := src.(source.Watcher); ok {
    go w.Watch(ctx, events)
    handleEvents(ctx, pool, pCfg, events)
} else {
    fmt.Println("  source does not support watching — use fsvector reconcile")
    // block on context cancellation only
    <-ctx.Done()
}
```

## LocalSource

Thin adapter over existing `fsindex` and `watcher` packages.
Neither package changes.
```go
// source/local.go
type LocalSource struct {
    Root string
}

func (s *LocalSource) Walk(ctx context.Context) ([]FileInfo, error) {
    files, err := fsindex.Walk(s.Root)
    if err != nil {
        return nil, err
    }
    return convertFileInfos(files), nil
}

func (s *LocalSource) Reader() FileReader { return &LocalReader{} }
func (s *LocalSource) URI() string        { return "local" }

// Watch implements Watcher
func (s *LocalSource) Watch(ctx context.Context, events chan<- Event) error {
    // wraps existing watcher.Watch, converts watcher.Event → source.Event
}
```

`fsindex` and `watcher` packages are completely unchanged.

## Pipeline Changes

`pipeline.Config` gains a `Reader` field:
```go
type Config struct {
    Reader          source.FileReader  // replaces os.ReadFile
    EmbedClient     *embed.Client
    ConvertClient   *convert.Client
    TranscribeClient *transcribe.Client
    VisionClient    *vision.Client
    EmbedModel      string
    Source          string
    MinEmbedSize    int64
    ChunkSize       int
    ChunkOverlap    int
    MinChunkSize    int
    VideoFrameRate  float64
    CaptionFrames   bool
}
```

`readFile` in `pipeline/pipeline.go` becomes:
```go
func readFile(ctx context.Context, cfg Config, path string) ([]byte, error) {
    return cfg.Reader.Read(ctx, path)
}
```

All process functions receive `ctx` and use `cfg.Reader.Read` instead
of `os.ReadFile`. This is the only pipeline change — all other logic
is identical for local and S3 files.

## Daemon Package

`internal/daemon/` owns the high-level orchestration and HTTP API.
`cmd/fsvectord/main.go` becomes a thin entrypoint that constructs
dependencies and calls `daemon.Run`.
```
internal/daemon/
├── daemon.go      # Run() — wires reconcile + watch + server
├── reconcile.go   # reconciliation logic (moved from fsvectord/main.go)
└── server.go      # local HTTP API
```

### Why daemon/ combines reconcile and server

Reconciliation and the HTTP server are tightly coupled:
- `POST /reconcile` triggers reconciliation
- `GET /reconcile/status` queries reconciliation progress
- Server needs access to reconciliation state (running, progress, errors)
- Both share the same pool, source, and pipeline config

Separating them would require passing state between packages. Keeping
them in `daemon/` with shared state is cleaner.

### daemon.go
```go
package daemon

type Daemon struct {
    pool   *pgxpool.Pool
    source source.Source
    pCfg   pipeline.Config
    server *Server
    port   int
}

func New(pool *pgxpool.Pool, src source.Source, pCfg pipeline.Config, port int) *Daemon

func (d *Daemon) Run(ctx context.Context) error {
    // 1. reconcile on startup
    if err := d.Reconcile(ctx); err != nil {
        return err
    }

    // 2. start HTTP server
    go d.server.Serve(ctx, d.port)

    // 3. watch if supported
    if w, ok := d.source.(source.Watcher); ok {
        events := make(chan source.Event, 64)
        go w.Watch(ctx, events)
        d.handleEvents(ctx, events)
    } else {
        <-ctx.Done()
    }
    return nil
}
```

### server.go — HTTP API
```go
// Endpoints:
// POST /reconcile          → trigger reconciliation, returns {"job_id": "..."}
// GET  /reconcile/status   → {"status": "running|idle", "progress": {...}}
// GET  /status             → daemon health, source, last reconcile time
// GET  /health             → {"status": "ok"}
```

Port configurable via `DAEMON_PORT` env var, default `8080`.
Binds to `127.0.0.1` only — not exposed outside localhost.

### reconcile.go

Reconciliation logic moved verbatim from `cmd/fsvectord/main.go`.
Progress tracked in a shared struct accessible to the server:
```go
type ReconcileProgress struct {
    Running   bool
    Total     int
    Indexed   int
    Deleted   int
    Skipped   int
    Errors    []string
    StartedAt time.Time
    FinishedAt *time.Time
}
```

## fsvector reconcile command
```go
// cmd/fsvector/main.go
var reconcileCmd = &cobra.Command{
    Use:   "reconcile",
    Short: "Trigger a full reconciliation on the daemon",
    RunE: func(cmd *cobra.Command, args []string) error {
        // POST to daemon HTTP server
        resp, err := http.Post(
            fmt.Sprintf("http://127.0.0.1:%d/reconcile", daemonPort),
            "application/json", nil)
        // stream progress until done
    },
}
```

New flag:
```
--daemon-port  int  port fsvectord is listening on (default 8080)
```

Or read from `DAEMON_PORT` env var.
