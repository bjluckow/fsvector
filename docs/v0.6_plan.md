# fsvector v0.6 — Vision (OCR + Captioning)

## Overview

Adds `visionsvc` — a new Python/FastAPI service wrapping Tesseract (OCR)
and BLIP (image captioning). Vision processing applies to image files and
video frames, making visual content fully searchable via both semantic
(CLIP embedding) and keyword (FTS on caption + OCR text) search.

Audio files are unchanged.

---

## What visionsvc Does
```
POST /caption   (image bytes) → {"caption": "a brown dog running on grass"}
POST /ocr       (image bytes) → {"text": "WARNING: wet floor\nExit this way"}
GET  /health                  → {"status": "ok", "caption_model": "...", "ocr": true}
```

Two separate endpoints — caption and OCR are independent operations.
Both accept raw image bytes as multipart upload.

---

## Models

| Task | Model | Size | Notes |
|---|---|---|---|
| Captioning | `Salesforce/blip-image-captioning-base` | ~900MB | Good quality, CPU-capable |
| OCR | Tesseract | ~50MB | System package, no GPU needed |

Both loaded on startup. Configurable via env:
```
CAPTION_MODEL=Salesforce/blip-image-captioning-base
VISION_OCR_ENABLED=true
```

### `main.py`
```python
import os
import io
from fastapi import FastAPI, UploadFile, File, HTTPException
from PIL import Image
import pytesseract
from transformers import BlipProcessor, BlipForConditionalGeneration
import torch

CAPTION_MODEL = os.environ.get("CAPTION_MODEL", "Salesforce/blip-image-captioning-base")
OCR_ENABLED = os.environ.get("VISION_OCR_ENABLED", "true").lower() == "true"

print(f"loading caption model: {CAPTION_MODEL}")
processor = BlipProcessor.from_pretrained(CAPTION_MODEL)
model = BlipForConditionalGeneration.from_pretrained(CAPTION_MODEL)
model.eval()
print(f"caption model loaded")

app = FastAPI()

@app.get("/health")
def health():
    return {
        "status": "ok",
        "caption_model": CAPTION_MODEL,
        "ocr": OCR_ENABLED,
    }

@app.post("/caption")
async def caption(file: UploadFile = File(...)):
    contents = await file.read()
    image = Image.open(io.BytesIO(contents)).convert("RGB")
    inputs = processor(image, return_tensors="pt")
    with torch.no_grad():
        out = model.generate(**inputs, max_new_tokens=50)
    text = processor.decode(out[0], skip_special_tokens=True)
    return {"caption": text.strip()}

@app.post("/ocr")
async def ocr(file: UploadFile = File(...)):
    if not OCR_ENABLED:
        raise HTTPException(status_code=503, detail="OCR is disabled")
    contents = await file.read()
    image = Image.open(io.BytesIO(contents)).convert("RGB")
    text = pytesseract.image_to_string(image).strip()
    return {"text": text}
```

## DB Storage Pattern

### Images
```
image.jpg
  chunk_index=0  chunk_type=NULL   embedding=CLIP   text_content="a brown dog on grass"
  chunk_index=1  chunk_type='ocr'  embedding=NULL   text_content="WARNING: wet floor"
  chunk_index=2  chunk_type='ocr'  embedding=NULL   text_content="..."
```

- `chunk_index=0` — the primary image row. CLIP embedding for visual
  search. Caption stored as `text_content` for FTS. Always present.
- `chunk_index=1..N` — OCR rows. No embedding. `text_content` is OCR
  chunk text. Only present if Tesseract finds text in the image.

### Video frames
```
video.mp4
  chunk_index=0   chunk_type='frame'       embedding=CLIP  text_content="a man in a suit"
  chunk_index=1   chunk_type='frame'       embedding=CLIP  text_content="a white wall"
  chunk_index=N   chunk_type='transcript'  embedding=text  text_content="..."
```

- Each frame row gets a caption as `text_content`. No OCR on frames.
- Transcript rows unchanged.

### Audio

Unchanged — no vision processing.

### image.go — describeImage

New shared helper used by both `processImage` and the frame loop in
`video.go`:
```go
// describeImage calls visionsvc to get a caption and runs OCR.
// Returns the caption as text_content and OCR chunks as additional files.
// chunkOffset is the chunk_index to start OCR rows from.
func describeImage(
    ctx context.Context,
    cfg Config,
    fi fsindex.FileInfo,
    imageData []byte,
    embeddingChunkIndex int,
    chunkOffset int,
) (captionText string, ocrFiles []store.File, err error) {

    // 1. caption
    capResp, err := cfg.VisionClient.Caption(ctx, fi.Name, imageData)
    if err != nil {
        return "", nil, fmt.Errorf("caption %s: %w", fi.Path, err)
    }
    captionText = capResp.Caption

    // 2. OCR
    ocrResp, err := cfg.VisionClient.OCR(ctx, fi.Name, imageData)
    if err != nil || strings.TrimSpace(ocrResp.Text) == "" {
        return captionText, nil, nil
    }

    // 3. chunk OCR text
    chunks := chunk.Split(ocrResp.Text, cfg.ChunkSize, cfg.ChunkOverlap, cfg.MinChunkSize)
    ocrType := "ocr"
    for i, c := range chunks {
        f, err := processTextChunk(ctx, cfg, fi, c, chunkOffset+i)
        if err != nil || f == nil {
            continue
        }
        f.Modality = fi.Ext  // preserve original modality (image/video)
        f.ChunkType = &ocrType
        f.Embedding = nil    // OCR rows have no visual embedding
        ocrFiles = append(ocrFiles, *f)
    }

    return captionText, ocrFiles, nil
}
```

### image.go — processImage updated
```go
func processImage(ctx context.Context, cfg Config, fi fsindex.FileInfo) (Result, error) {
    data, err := readFile(fi.Path)
    if err != nil {
        return Result{}, fmt.Errorf("read %s: %w", fi.Path, err)
    }

    // convert to JPEG if needed
    if fi.Ext != "jpg" && fi.Ext != "jpeg" {
        data, err = cfg.ConvertClient.ConvertToImage(ctx, fi.Name, data)
        if err != nil {
            return Result{}, fmt.Errorf("convert %s: %w", fi.Path, err)
        }
    }

    // CLIP embedding
    vector, err := cfg.EmbedClient.EmbedImage(ctx, fi.Name, data)
    if err != nil {
        return Result{}, fmt.Errorf("embed %s: %w", fi.Path, err)
    }

    // vision: caption + OCR
    captionText, ocrFiles, err := describeImage(ctx, cfg, fi, data, 0, 1)
    if err != nil {
        // non-fatal — log and continue without caption/OCR
        fmt.Fprintf(os.Stderr, "    vision %s: %v\n", fi.Path, err)
    }

    primaryRow := store.File{
        Path:           fi.Path,
        Source:         cfg.Source,
        ContentHash:    fi.Hash,
        Size:           fi.Size,
        MimeType:       fi.MimeType,
        Modality:       "image",
        FileName:       fi.Name,
        FileExt:        fi.Ext,
        FileCreatedAt:  &fi.CreatedAt,
        FileModifiedAt: &fi.ModifiedAt,
        EmbedModel:     cfg.EmbedModel,
        Embedding:      vector,
        ChunkIndex:     0,
        TextContent:    &captionText,
    }

    files := append([]store.File{primaryRow}, ocrFiles...)
    return Result{Files: files}, nil
}
```

## Search

No changes needed. Image rows now have `text_content` (caption) so
they participate in hybrid FTS automatically. OCR rows have
`text_content` and `embedding=NULL` — they appear in FTS results
but not in pure semantic search since `embedding IS NOT NULL` filter
excludes them.

This means searching "dog" returns:
- Image rows whose CLIP embedding is close to "dog" (semantic)
- Image rows whose caption contains "dog" (FTS boost)
- OCR rows whose extracted text contains "dog" (FTS boost)
