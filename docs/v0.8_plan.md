# fsvector v0.8 — Daemon API & CLI as HTTP Client

## Overview

v0.8 moves all query logic out of `cmd/fsvector` and into the daemon
HTTP API. `fsvector` CLI becomes a thin HTTP client — no database
connection, no pgx dependency, no DATABASE_URL required. The daemon
becomes the single access point for all fsvector functionality,
enabling plugins, a future web UI, and remote daemon access.

---

## Motivation

**Current architecture:**
```
fsvector CLI → postgres (direct)
fsvectord   → postgres (direct)
```

**v0.8 architecture:**
```
fsvector CLI → fsvectord daemon API → postgres
web UI       → fsvectord daemon API → postgres
plugins      → fsvectord daemon API → postgres
```

Benefits:
- `fsvector` needs no DB credentials — just a daemon URL
- All business logic lives in one place
- Web UI and plugins reuse the same API
- Remote daemon support — point CLI at a NAS or server
- Plugin API for tools like emailsorter

## Daemon API

All endpoints bind to `127.0.0.1` by default. Configurable via
`DAEMON_HOST` or `daemon.host` in `fsvector.yaml` for remote access.

### Plugin API
```
POST /embed/text
{"texts": ["a photo of a dog"]}
→ {"embeddings": [[0.12, 0.84, ...]]}

POST /embed/image
(multipart: file=<image bytes>)
→ {"embedding": [0.12, 0.84, ...]}

GET /export/files
    ?modality=email&format=ndjson
→ NDJSON stream of file rows with embeddings
   one JSON object per line

POST /import/files
(NDJSON body)
→ {"imported": 42, "errors": 0}
```

`/embed/*` endpoints are passthroughs to embedsvc — plugins get
embeddings without knowing about embedsvc directly.

`/export` and `/import` enable cross-instance sync and plugin
databases. Each line in the NDJSON export is a complete file row
including the embedding vector.

## fsvector CLI — HTTP Client

`fsvector` drops all direct DB and embed service dependencies.
It becomes a pure HTTP client against the daemon API.

### Root command — --host flag
```go
var daemonHost string

func init() {
    rootCmd.PersistentFlags().StringVarP(
        &daemonHost, "host", "H",
        "http://localhost:8080",
        "fsvectord daemon address",
    )
    viper.BindPFlag("daemon.host", rootCmd.PersistentFlags().Lookup("host"))
    viper.BindEnv("daemon.host", "DAEMON_HOST")
    viper.SetDefault("daemon.host", "http://localhost:8080")
}
```

### Dependencies removed from fsvector

- `github.com/jackc/pgx/v5`
- `github.com/pgvector/pgvector-go`
- `internal/search/`
- `internal/store/`
- `internal/clients/embed`
- `internal/config` (replaced by simple host flag + viper)

## daemon/server.go — new endpoints

Add to existing server:
```go
mux.HandleFunc("/search", s.handleSearch)
mux.HandleFunc("/files", s.handleFiles)
mux.HandleFunc("/files/", s.handleFileDetail)
mux.HandleFunc("/stats", s.handleStats)
mux.HandleFunc("/embed/text", s.handleEmbedText)
mux.HandleFunc("/embed/image", s.handleEmbedImage)
mux.HandleFunc("/export/files", s.handleExportFiles)
mux.HandleFunc("/import/files", s.handleImportFiles)
```

The server needs access to the DB pool and embed clients.
Update `Server` struct:
```go
type Server struct {
    pool        *pgxpool.Pool
    embedClient *embed.Client
    progress    *Progress
    trigger     chan struct{}
    started     time.Time
    source      string
}
```

### Search handler
```go
func (s *Server) handleSearch(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "method not allowed", http.StatusMethodNotAllowed)
        return
    }

    var req SearchRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }

    // embed query text
    vectors, err := s.embedClient.EmbedTexts(r.Context(), []string{req.Query})
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    // parse filters
    q, err := buildSearchQuery(req, vectors[0])
    if err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }

    results, err := search.Search(r.Context(), s.pool, q)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    results = search.Normalize(results)
    json.NewEncoder(w).Encode(SearchResponse{Results: results})
}
```

### M1.7.4 — fsvector CLI as HTTP client
**Goal:** All `fsvector` subcommands use `apiclient` instead of
direct DB/embed connections. `--host`/`-H` flag on root command.
`DATABASE_URL` no longer required by `fsvector`.

**Verify:**
```bash
# default host
fsvector search "dog"
fsvector ls
fsvector stats
fsvector show "/path/to/file"

# explicit host
fsvector --host http://localhost:8080 search "dog"
fsvector -H http://localhost:8080 stats

# env var
DAEMON_HOST=http://localhost:8080 fsvector search "dog"

# no DATABASE_URL needed
unset DATABASE_URL
fsvector search "dog"
# → works fine
```
